Point-wise SFT — Yes/No Click Prediction

Qwen3-0.6B / 1.7B · Discover-feed candidate ranking · v10 splits · ctx 4096
  • Task. Each (user, candidate) pair becomes a single-token binary classification: continue the prompt with " Yes" or " No"; label = is_clicked. At inference, ranking score is P(" Yes") / (P(" Yes") + P(" No")).
  • Prompt. Chat-template-wrapped system + user with last 30 click history items, the interests blob, and the candidate (title / summary). enable_thinking = false, cutoff 4096.
  • Loss / Optim. Weighted cross-entropy on the answer token only (prompt tokens masked with -100). AdamW-8bit, lr 2e-5, bs 128 (mbs 2 + grad-accum), 2 epochs, neg_frac 0.3, DeepSpeed ZeRO across 8×A100.

1. Raw row

{
  user_id, bizdate,
  history: [...30],
  interests: {...},
  candidate: {title, summary},
  label: 0 / 1
}
→

2. Prompt build

Chat template, system + user concatenated: 
SYS: You are a ranker...
USR: History: ...
     Interests: ...
     Candidate: title / summary
     Will the user click?
ASSISTANT:
→

3. Label mask

Tokens:
[SYS ... USR ... ASSISTANT:] " Yes" or " No"

Masked tokens carry label = -100 — only the final answer token contributes to CE.
→

4. Weighted CE

logits = LM(input_ids)
ce     = CE(logit[ans], "Yes" / "No")
loss   = (ce * weight).mean()
Memory-efficient: gather logits only at active positions.
→

5. Inference

Read next-token logits:
p = softmax([l_Yes, l_No])[0]

p ∈ [0,1] → ranking score → AUC.

Loss

$$L = \frac{1}{N}\sum_i w_i \cdot \mathrm{CE}\bigl(z_i^{\text{ans}},\; y_i \in \{\text{Yes},\text{No}\}\bigr)$$ Prompt-segment tokens have label = -100 and do not contribute to gradients.
